Q1: windows 提供的 wsl 算容器還是vm, 還是算cli而已，有點難以理解其定位

A1:
	WSL是什麼?
		全名 windows Subsystem for Linux
		在 windows 環境下直接操作 linux ，不需要雙啟動或使用虛擬機

	定位:
		WSL 的定位不是單一概念，而是「隨版本演進、介於 VM、容器與 CLI 之間的一種特殊存在」。

	結論
		WSL ≠ 容器、≠ 傳統 VM、≠ 單純 CLI
		👉 WSL2 本質上是「輕量化 VM + 深度整合 Windows」
		👉 WSL1 則是「系統呼叫轉譯層（Compatibility Layer）」


	WSL 的兩個世代差異（這是關鍵）
		🔹 WSL 1（舊，現在幾乎不用）
			本質：系統呼叫轉譯（不是 VM）
				Linux App
				   ↓
				WSL syscall translation
				   ↓
				Windows NT Kernel

			特點：
				❌ 沒有 Linux kernel

				❌ 不能跑 Docker（沒有 cgroups / namespaces）

				✅ 超快啟動

				❌ 相容性有限（很多 Linux 功能不支援）

			👉 這一代「有點像 CLI + 相容層」

		🔹 WSL 2（現在主流）
			本質：真正的 Linux Kernel + 輕量 VM
				Linux App
				   ↓
				Real Linux Kernel
				   ↓
				Hyper-V (lightweight VM)
				   ↓
				Windows

			特點：
				✅ 真正 Linux Kernel（微軟自己維護）

				✅ 支援 Docker、K8s、iptables、cgroups

				✅ 幾乎 100% Linux 相容

				❌ 本質上「確實是 VM」

			👉 這一代：技術上是 VM，但「使用體驗不像 VM」

	那它到底是不是 VM？
		技術上：✅ 是 VM
		使用體驗：❌ 不像傳統 VM

			| 項目                     | 傳統 VM            | WSL2                 |
			| ------------------------ | ------------------ | -------------------- |
			| 是否有 Linux Kernel      | ✅                 | ✅                    |
			| 是否使用 Hypervisor      | ✅                 | ✅（Hyper-V）         |
			| 是否要開一台完整 OS      | ✅                 | ❌                    |
			| 是否需要手動開機         | ✅                 | ❌                    |
			| 是否和 Windows 共用檔案  | ❌                 | ✅                    |
			| IP 是否獨立              | ✅                 | ⚠️ 半獨立             |
			| 使用者感覺               | 「我在開一台機器」 | 「我在開一個 shell」 |

		👉 WSL2 是「被極度弱化存在感的 VM」


	那它算不算容器？
		❌ 不是容器（但很常被誤會）

		關鍵差異
			| 項目         | 容器 (Docker)        | WSL2                  |
			| ------------ | -------------------- | --------------------- |
			| Kernel       | 共用宿主 Kernel      | **獨立 Linux Kernel** |
			| 隔離方式     | namespaces + cgroups | **Hyper-V VM**        |
			| 啟動速度     | 非常快               | 快，但仍是 VM         |
			| OS 概念      | 無完整 OS            | 有完整 Linux          |
			| systemd      | ❌（預設）           | ✅                     |
			| 適合跑多服務 | ⚠️                   | ✅                     |

		👉 WSL2 裡面可以跑 Docker，但 WSL 本身不是 Docker


	那它只是 CLI 嗎？
		❌ 不只是 CLI

		雖然你用起來像這樣：
			wsl
			ubuntu

		但實際上你是在：

			* 進入一個完整 Linux OS

			* 有自己的 /proc、/sys

			* 有自己的 systemd（新版已支援）

			* 可以跑 database、message queue、docker daemon

		👉 CLI 只是入口，不是本體

	正確的定位說法
		WSL2 是：

			* 一個「為了開發者設計」

			* 「深度整合 Windows 的 Linux VM」

			* 「讓你感覺不像在用 VM 的 VM」

		或一句工程師版：

			WSL2 = Developer-friendly Linux VM runtime


	為什麼微軟不直接叫它 VM？
		因為它的目標不是：

			* 跑 Production

			* 管 VM lifecycle

			* 管網路拓樸

		而是：

			* 開發

			* Build

			* CLI tooling

			* Docker / K8s 本機環境

		👉 定位是「開發基礎設施」，不是虛擬化產品

	你可以這樣記（很實用）

		❌ 把 WSL 當 Docker → 會誤解隔離模型

		❌ 把 WSL 當 VM → 會誤解使用方式

		❌ 把 WSL 當 CLI → 會誤解能力

		✅ 正確心智模型

			「我在 Windows 裡，隨時開一個 Linux 世界」
