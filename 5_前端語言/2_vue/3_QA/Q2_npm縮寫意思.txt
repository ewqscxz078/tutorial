Q1:	npm 原本的縮寫是：Node Package Manager
		不過在官方現在的說法裡，npm 已經被當成專有名詞，不再強調一定要展開成全名（就像 Git、Linux 一樣），但它的歷史來源確實是這個意思。

A1: npm 到底是什麼？用工程師角度來看
	把 npm 理解成三個東西合在一起：
		1.套件管理工具（Package Manager）
			* 負責：

				* 下載套件（例如 vue, axios）

				* 管理版本（semver）

				* 處理相依關係樹（dependencies graph）

			* 類似你在 Java 世界的：

				* Maven / Gradle

		2.CLI 指令工具（npm command）
			在終端機打的：
				npm install
				npm run dev
				npm create vue@latest

			這些都是 npm 這個 CLI 在執行的動作。

		3.套件註冊中心（npm registry）
			* 預設是：https://registry.npmjs.org

			* 放了上百萬個 JavaScript / TypeScript 套件

			* 類似：

				* Maven Central

	熟悉的東西對照（很重要）
		| Java / 後端世界    | Node / 前端世界         |
		| ------------------ | ----------------------- |
		| Java               | JavaScript / TypeScript |
		| JDK                | Node.js                 |
		| Maven / Gradle     | npm / pnpm / yarn       |
		| pom.xml            | package.json            |
		| mvn install        | npm install             |
		| local .m2 repo     | node_modules（專案內）  |
		| Maven Central      | npm registry            |


	那為什麼現在還是叫 npm？

		* 歷史原因 + 生態系慣性：

		* npm 最早是 Node.js 專用套件管理器

		* 後來前端（Vue / React / Angular）全面依賴 npm 生態

		* 即使現在：

			* 可用 pnpm / yarn

			* 可管理「非 Node runtime 的前端專案」

		* 名字就沿用到現在

	一個你可能會在意的點（工程實務）
		npm 現在不只是「裝套件」，它還負責：

			* 執行專案腳本（npm run build）

			* 專案初始化（npm create ...）

			* 管理 lock file（package-lock.json）

		這點其實跟 Maven 越來越像。

	總結一句話

		npm = Node 生態系的 Maven + CLI 工具 + 中央套件庫


	你可以用這幾個指令更直觀看它在做什麼
		1) 看它其實等同於跑 create-vue
			npm create vue@latest
			# ≈
			npm init vue@latest

		2) 直接用 npx 跑（概念更明顯）
			npx create-vue@latest

			// 這就很直白：用 npx 下載/執行某個套件的 CLI。


	重要補充：它會不會「真的打網路」？
		* 第一次通常會去 npm registry 抓 create-vue（除非你已有快取或公司有內部 registry）

		* 建專案本身是本機寫檔

		* npm install 才是大量下載依賴套件的主要階段
