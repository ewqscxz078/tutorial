20251231 當時使用「最新的 Vue」開始做前端開發，建議直接走 
	Vue 3 + Vite + TypeScript 這條主流路線（也是目前最常見、最順的組合）
	
	ref https://nodejs.org/zh-tw/download
	
0) 先準備環境（Windows 也適用）
	1.安裝 Node.js LTS（建議用 LTS，不要追最新 Current）

	2.打開終端機確認版本：
		node -v
		npm -v

1) 用官方方式建立 Vue 專案（Vite）
	npm create vue@latest
		* npm create 是 npm 提供的「建立專案腳手架（scaffold）」入口（等同於以前常見的 npm init <initializer> 形式）
		* vue@latest 指的是：使用 npm registry 上的 create-vue 這個「專案初始化器套件」的最新版本來幫你生專案

		換句話說，它做的是：

			1.到 npm registry 下載（或用快取）一個叫 create-vue 的套件（版本是 latest）

			2.在你本機執行這個套件的 CLI 程式

			3.依你回答的選項，把一個 Vue + Vite 專案骨架「寫到你的資料夾」裡

			4.它不會幫你自動 npm install（有些會問要不要幫你跑），你通常自己再裝依賴、啟動 dev server

		所以它比較像：
			*「下載一個腳手架程式 → 本機執行 → 產生專案模板」
			* 而不是：「拉一個遠端打包好的完整環境（像 Docker image）」

	它會問要不要開啟一些選項，我推薦你勾這些（很適合工程化）：

		* ✅ TypeScript
		* ✅ Vue Router（前端路由）
		* ✅ Pinia（狀態管理）
		* ✅ ESLint（程式碼規範）
		* ✅ Prettier（格式化）

	然後進專案、裝依賴、跑起來：
		cd your-project-name
		npm install
		npm run dev

	看到終端機提示的網址（通常是 http://localhost:5173）就成功了。


2) Vue 專案基本結構你要先懂的幾個重點
	會常碰到這些檔案/資料夾：

		src/main.ts：入口，掛載 Vue App

		src/App.vue：根元件

		src/router/index.ts：前端路由表（像你以前 JSP 不同頁面，但這裡是 SPA）

		src/stores/*：Pinia stores（類似你以前 session 裡放一些狀態，但更乾淨）

		src/components/*：可重用元件（表格、按鈕、Modal）

		src/views/*：頁面級元件（例如 UserListView）

3) 你以前用 JSP/JSF 最常問的：Vue 有「常用元件」嗎？

	有，而且現在通常不是 WebJar 那種模式，而是走：

		* UI 元件庫：例如 Element Plus、Naive UI、Vuetify（直接裝 npm 套件）

		* Headless + 自由組裝：例如 Tailwind + Headless UI / Radix Vue

		* 企業常用：Element Plus（上手快，表單/表格一大堆現成）

	如果你要我用「最少決策成本」給你一套，我會建議：

		* 想快、元件齊：Element Plus

		* 想設計感、客製化彈性：Tailwind + shadcn-vue 類型方案（或 Radix Vue）


4) 前後端分離怎麼接（你是後端工程師這段很關鍵）

	一般做法是：

		* Vue 只負責呼叫後端 API（JSON）

		* 後端（Spring Boot）提供 API + OAuth2/JWT 驗證

		* 本機開發用 Vite proxy 解決 CORS / cookie / token 測試
		
	Vite 代理（超常用）
		在 vite.config.ts 加類似：
			export default defineConfig({
			  server: {
				proxy: {
				  "/api": {
					target: "http://localhost:8080",
					changeOrigin: true,
				  }
				}
			  }
			})
		之後你前端打 /api/... 就會被轉到後端 8080

5) 建議用的學習路線（3～5 天就能上手到能做功能）

	* Vue SFC + <script setup>（最常用寫法）

	* Props / Emits / v-model（元件資料流）

	* Vue Router（頁面切換、守衛）

	* Pinia（登入狀態、使用者資料）

	* 呼叫 API：axios / fetch（含攔截器、JWT）

	* 表單驗證（元件庫通常自帶）

6) 一個「現在就可以做」的小練習（最有感）

	做一個登入頁 + 呼叫 /api/me：

		* 登入成功把 access token 放記憶體或 localStorage（依你安全需求）

		* axios interceptor 自動加 Authorization: Bearer ...

		* 進 /profile 顯示使用者資料

	這一套練完，你就真的可以開始切你們系統的頁面了。
